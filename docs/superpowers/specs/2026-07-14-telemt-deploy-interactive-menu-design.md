# telemt-deploy — интерактивное главное меню

**Дата:** 2026-07-14  
**Статус:** Утверждено пользователем  
**Версия целевая:** 2.4

## Цель

Превратить `install.sh` в интерактивную панель управления: при запуске без флагов открывается главное меню (как MEKO Launcher), из которого доступны установка, статистика, мониторинг, сервисы, настройки, SSL, MEKO, firewall, проверки, обновление telemt и удаление стека.

Флаги CLI (`--domain`, `--fresh`, `--status`, …) сохраняются для автоматизации и обходят меню.

## Контекст

Текущий `telemt-deploy` — линейный bash-установщик с модулями в `lib/`:

- `install.sh` — роутер и установка
- `lib/env.sh` — обнаружение существующей установки
- `lib/stats.sh` — статистика (MEKO-подсчёт людей по `active-ips`, TCP из `/v1/users`)
- `lib/verify.sh`, `lib/handoff.sh`, `lib/telemt.sh`, … — установка и проверки

Проблемы текущего UX: установка линейна, статистика только через `--status`, интерактивность нестабильна без единой точки входа.

## Решения (утверждено)

| Вопрос | Решение |
|--------|---------|
| Точка входа | **B** — `install.sh` без флагов → главное меню |
| Охват функций | **C** — полное управление стеком |
| Статистика | **C** — разовый снимок + live-мониторинг |
| UI | **C** — текстовое меню + `dialog` для опасных действий (fallback на `[y/N]`) |
| Удаление | Отдельный пункт **11** главного меню |

## Архитектура

### Роутинг `install.sh`

```
install.sh [флаги]
    │
    ├─ есть action-флаги (--status, --uninstall, --domain, …)
    │       └─► текущий non-interactive flow (без меню)
    │
    ├─ не TTY / pipe
    │       └─► ошибка с подсказкой использовать флаги
    │
    └─ интерактивный TTY, без action-флагов
            └─► main_menu() [цикл]
```

**Action-флаги** (обход меню): `--status`, `--uninstall`, `--domain`, `--ad-tag`, `--telemt-version`, `--meko-full`, `--yes`, `--fresh`, `--keep`, `--reinstall`.

Запуск `install.sh` без аргументов → только меню.

### Новые модули

| Файл | Ответственность |
|------|-----------------|
| `lib/menu.sh` | Главное меню, подменю, роутинг пунктов |
| `lib/monitor.sh` | Live-мониторинг (цикл обновления) |
| `lib/dialog.sh` | Обёртка `dialog`/`whiptail` + текстовый fallback |
| `lib/stats.sh` | Расширение: шапка меню, список IP, форматирование |
| `install.sh` | Тонкий роутер: флаги vs `main_menu` |

Существующие `lib/*.sh` переиспользуются; логика установки не дублируется.

## Главное меню

Шапка (обновляется при каждом отображении):

- Версия installer (`2.4`)
- Домен:порт из state / `telemt.toml`
- Версия telemt
- **Подключено: N человек** (жёлтый, MEKO-алгоритм)
- TCP-соединений
- Статусы: `telemt`, `nginx`, `mtpr-synfix` (OK/FAIL)

Пункты:

1. Установка / переустановка  
2. Статистика (разово)  
3. Мониторинг (live)  
4. Сервисы (старт / стоп / рестарт / логи)  
5. Настройки прокси (домен, ad_tag, секрет, ссылка)  
6. SSL / сертификат  
7. MEKO SYN FIX  
8. Firewall (UFW)  
9. Проверки (DNS, 443, mask-site)  
10. Обновить telemt  
11. Удалить стек  
0. Выход  

Пункты 2–11 при отсутствии установки показывают предупреждение «сначала выполните установку» (кроме 1 и 11).

## Подменю (кратко)

### 1) Установка / переустановка

Вызывает существующий flow: `handle_existing_env` → домен → DNS → `prereq` … → `verify` → handoff → `ad_tag`. Логика из текущего `install.sh` выносится в `run_install_flow()`.

### 2) Статистика (разово)

Экран: люди (жёлтым), TCP, трафик (`total_octets` если доступен), ссылка, список IP из `/v1/stats/users/active-ips`. Enter → назад.

### 3) Мониторинг (live)

`clear` + шапка каждые 4 секунды. Выход: `q` или `0`. Не блокирует ввод в отдельном потоке — опрос клавиатуры через `read -t` или `stty` + прерывание цикла.

### 4) Сервисы

Подменю: telemt / nginx / mtpr-synfix → status | restart | stop | start | journalctl (50 строк).

### 5) Настройки прокси

- Просмотр секрета и ссылки  
- Ввод/смена `ad_tag` → `telemt_write_config` + restart  
- Просмотр `telemt.toml` (первые N строк или `less`)

### 6) SSL

- Статус certbot / путь к сертификату  
- `certbot renew --dry-run` или renew  
- Дата истечения

### 7) MEKO SYN FIX

- Статус цепочки `MTPR_SYNFIX`  
- Переустановка inline rules (`meko_install_inline`)  
- Информация о `--meko-full` (без автопереключения без confirm)

### 8) Firewall

- `ufw status`  
- Показать правила 80/443  
- Переприменить `firewall_setup`

### 9) Проверки

Вызов `verify_install "$DOMAIN"` без переустановки.

### 10) Обновить telemt

- Показать текущую и latest версию  
- Подтверждение → `telemt_install_binary` + restart

### 11) Удалить стек

`dialog` confirm (или `[y/N]`) → `uninstall_all`. После удаления — сообщение и возврат в меню.

## UI-слой (гибрид)

- **Меню, мониторинг, статистика:** `echo` + `prompt_line` / `prompt_choice` через `/dev/tty`  
- **Опасные действия:** `confirm_dialog "Текст"` → `dialog --yesno` если `command -v dialog`; иначе `confirm_yes`  
- **Цвета:** `YELLOW` — число людей; `GREEN` — OK; `RED` — ошибки; `CYAN` — метки  

`prereq_install` дополнить опциональной установкой `dialog` (не обязательна).

## Данные и состояние

При входе в меню и при каждой отрисовке шапки:

1. `env_load_settings` из `lib/env.sh`  
2. API telemt `127.0.0.1:9091` (только localhost)  
3. `systemctl is-active` для сервисов  

Файлы: `/root/telemt-deploy.state`, `/root/telemt-secret.txt`, `/etc/telemt/telemt.toml`.

## Обработка ошибок

- `set +e` внутри цикла меню; ошибки показываются, меню не падает молча  
- API недоступен → `0 человек`, предупреждение `[!]`  
- Нет `dialog` → текстовые подтверждения  
- Не TTY → `die` с текстом «используйте флаги: --status, --domain …»

## Обратная совместимость

- `status.sh` → по-прежнему `install.sh --status`  
- Все существующие флаги работают без меню  
- `INSTALLER_VERSION` → `2.4`  
- Проверка bundle: `STATS_SH_VERSION`, `MENU_SH_VERSION`, …

## Тестирование

- `tests/smoke.sh` — синтаксис  
- Ручной чеклист: меню без установки, после установки, live monitor выход, uninstall п.11, флаги `--status` / `--domain --yes`
